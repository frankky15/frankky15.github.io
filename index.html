My Work

Retail Management Solution:

I found a sample database for a bike shop and decided an RMS web application would be a great use of it.
It is an integrated solution that adopts the Repository-Service and MVC Patterns, using Entity Framework with a custom implementation of users that binds to the sample database.
It implements roles, and each role is packed of features that are well documented in the instructions.
The app is divided into a Production section, a Sales section, and an Administration section. Some of the features of the production section are managing products, brands, categories, and updating stock.
The Sales section has features oriented towards managing orders and customer info.
As for the Administration section, it has features to manage staff members and each store location. So it also supports multiple locations.
Another great feature it has is a point of sale (POS) integration, so you can make orders straight from the app.
*In the future maybe i'll include some "dummy/mockup" api for the payment.*

Task Management System:

A SUPER simple Demo MVC web application I built to teach myself how to make a data access layer from scratch, and that was the challenging part. It has an in-memory database that uses JSON as storage.
It showed me how to properly handle errors in a very sensitive part of the code.
It adopts the Repository-Service Pattern and also has a very basic implementation of an authentication system that uses sessions.
In essence, it really showed me how a raw, well-structured application looks like.
